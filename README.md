# 小红书扫码发布 - 飞书多维表格字段捷径

一键生成小红书笔记发布二维码，扫码即可在小红书 APP 完成发布。

详细用户使用说明见：

- [../小红书扫码发布字段捷径-用户使用说明.md](../小红书扫码发布字段捷径-用户使用说明.md)

## 功能特性

- 📝 支持设置笔记标题（自动截取前20字）
- 📄 支持设置笔记正文内容
- 🖼️ 支持多图上传（首图为封面，其余为正文图片）
- 🎬 支持 1 个视频发布，可搭配 1 张封面图
- 🏷️ 支持添加标签（自动解析 # 分隔的标签）
- 📱 生成二维码，扫码在小红书APP完成发布

## 使用方法

### 1. 添加字段捷径

在多维表格中添加「小红书扫码发布」字段捷径。

### 2. 配置字段映射

| 配置项 | 说明 | 必填 |
|--------|------|------|
| 标题字段 | 选择包含笔记标题的文本字段 | ✅ |
| 正文字段 | 选择包含笔记正文的文本字段，可留空 | ❌ |
| 图片/视频字段 | 选择包含图片或视频的附件字段 | ✅ |
| 标签字段 | 选择包含标签的文本/多选字段 | ❌ |

### 3. 触发生成

填写数据后，插件会自动生成发布二维码，使用小红书 APP 扫码即可完成发布。

如当前插件已配置为付费插件且账号无剩余权益，字段捷径会返回付费提示，用户可按飞书插件中心的购买引导完成开通。

## 调试与发布注意事项

1. 本地调试建议配合飞书「字段捷径调试助手」使用
2. 飞书本地服务地址填写 `http://127.0.0.1:8080`
3. 调试助手会访问 `/open_field`、`/open_field_card` 等内部路径
4. `http://127.0.0.1:8080/` 根路径返回 `404 Not Found` 属于正常现象
5. 调试模式默认只处理第一行记录
6. 如修改了 `formItems` 或 `resultType`，需要重新添加字段后再验证
7. 正式发布使用 `npm run pack` 生成 zip 包，不要手工修改打包产物结构

## 字段输入格式

### 标题字段
普通文本，超过20字会自动截取。

### 正文字段
普通文本，支持换行，可留空。

### 图片/视频字段
附件字段，支持以下两种模式：

- 图片模式：
  - 支持 1-18 张图片
  - 第一张图片作为封面
  - 其余图片作为正文配图
  - 图片按文件名排序
- 视频模式：
  - 支持 1 个视频
  - 可额外搭配 0 张或 1 张图片作为封面
  - 不支持同时上传多个视频
  - 不支持 1 个视频搭配多张图片

### 标签字段
支持以下格式：
- 文本：`#标签1 #标签2 #标签3`
- 文本：`标签1,标签2,标签3`
- 多选字段：直接选择标签

## 返回结果

成功时返回小红书发布二维码图片（附件类型），用户扫码后在小红书APP完成发布。

## 错误处理

| 错误情况 | 处理方式 |
|----------|----------|
| 标题为空 | 返回错误，提示填写标题 |
| 正文为空 | 允许为空，不阻止生成 |
| 图片为空 | 返回错误，提示上传图片 |
| 网络异常 | 返回错误，提示稍后重试 |
| 额度用完 | 返回付费错误，提示购买 |

## 计费说明

本插件采用飞书插件中心官方付费能力。

1. 具体免费额度、按次计费规则以插件中心配置为准
2. 只有真正成功生成二维码并返回 `FieldCode.Success` 时才会消耗一次权益
3. 输入校验失败、无剩余权益、后端报错等场景不应计为成功扣次
